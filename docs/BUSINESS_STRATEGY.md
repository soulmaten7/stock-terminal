## 1. 서비스 정의

### 핵심 콘셉트
"전업투자자의 투자 환경을 일반 투자자에게 월 구독료로 제공하는 서비스"

### 문제 인식
전업투자자들은 투자할 때 수십 개의 창을 동시에 열어놓고 본다:
- 재무제표 (DART)
- 수급 데이터 (KRX)
- 공시 (DART)
- 공매도/신용잔고 (KRX)
- 내부자 거래
- 배당 데이터
- 뉴스 (여러 경제 매체)
- 섹터별 자금 흐름
- 거시경제 지표 (한국은행, FRED)
- 실시간 차트 (TradingView)

일반 투자자는 이 세팅을 따라하려면 수년이 걸리고, 어디서 뭘 봐야 하는지조차 모른다.

### 해결책
이 모든 데이터를 하나의 플랫폼에 통합해서 보여준다 — "미니 블룸버그 터미널" for 개인투자자.

### 핵심 가치
- **정보의 통합**: 흩어진 투자 정보를 한 곳에
- **접근성**: 전문가 수준의 데이터를 누구나 쉽게
- **시간 절약**: 세팅에 수년 걸리는 환경을 즉시 제공

## 2. 투자심사 Q&A (내부 검증 완료)

### 질문 1: 경쟁사 대비 차별점은?
**결론: 이 갭은 실재한다**
- 한국에 주식 관련 "모든 데이터"를 한 플랫폼에 모아놓은 서비스가 없음
- 네이버증권은 기본 데이터만, 증권사 HTS는 자사 고객만, 유료 서비스들은 특정 데이터만 제공
- StockTerminal은 공시+수급+공매도+내부자+배당+뉴스+거시경제+AI정리를 "전부" 통합
- 링크 허브까지 포함해서 투자자가 필요한 모든 외부 사이트도 카테고리별 정리

### 질문 2: 개발/운영 비용은 감당 가능한가?
**결론: 개발비 0원, 운영비 최소**
- 개발: Claude Code로 전체 개발 → 외주 비용 0원
- 인프라: Vercel(무료~$20/월) + Supabase(무료~$25/월) = 거의 0원으로 시작
- 데이터: DART, KRX, ECOS, FRED 전부 무료 공공 API
- AI: OpenAI GPT API만 유료 (종목당 분석 비용 매우 저렴)
- 자동화: Make(Integromat) 무료 플랜으로 시작 가능
- **기존 인프라 활용**: 창업자가 주식/가상화폐 광고주 허가 콜센터를 운영 중이라 광고주 네트워크 확보됨

### 질문 3: 규제/법적 리스크는?
**결론: "데이터 정리" 포지셔닝으로 회피**
- 핵심 결정: AI의 역할을 "주가 예측"이 아닌 "공개 데이터를 분석 기법별로 정리"로 설정
- 투자 자문이 아닌 "데이터 정리 서비스"이므로 금융감독원 규제 대상 아님
- 모든 페이지에 면책 조항 표시: "본 사이트는 투자 자문이 아니며, 투자 책임은 본인에게 있습니다"
- 배너 광고에도 면책 고지: "광고 배너는 광고주가 직접 등록한 것이며, 사이트는 광고 내용에 대한 책임을 지지 않습니다. 인증마크는 사업자등록 확인을 의미하며, 상품의 품질이나 수익을 보증하지 않습니다."

### 질문 4: 커뮤니티 관리는?
**결론: 플로팅 채팅 + 자동 모더레이션**
- 별도 커뮤니티 페이지를 만들면 채팅이 "너무 무거워"지고, 채팅방 링크 배너 목적과 충돌
- → 커뮤니티 페이지 삭제, 플로팅 채팅만 유지 (화면 좌하단)
- 금칙어 자동 필터 (투자 권유, 스팸 링크, 리딩방 홍보 등)
- 제재 시스템: 경고 → 30분 음소거 → 1개월 음소거
- 링크 입력 차단 (스팸 방지)
- 사업자 인증 안 된 광고성 글 차단

## 3. AI 전략 — "예측"이 아닌 "정리"

### 왜 이렇게 결정했나
- 창업자 인사이트: "AI가 판단해주는 걸 얼마나 믿겠어. 정보를 한눈에 보여주는 게 더 가치 있다"
- AI가 "이 종목 올라갑니다"라고 하면 → 틀리면 신뢰 상실, 법적 리스크
- AI가 "이 종목의 가치투자 관점 데이터 정리"라고 하면 → 틀릴 수가 없음 (사실만 정리)

### AI 분석 5가지 기법
1. **가치 분석**: PER, PBR, ROE, 부채비율 등 재무 데이터 정리
2. **기술적 분석**: 이동평균, RSI, MACD 등 기술적 지표 정리
3. **퀀트 분석**: 팩터 점수, 모멘텀, 밸류 팩터 정리
4. **배당 분석**: 배당수익률, 배당성향, 배당 히스토리 정리
5. **수급 분석**: 외국인/기관 매매 동향, 공매도 비율 정리

### AI 출력 형식
- "~입니다" 서술형 (자연어 요약)
- 절대 "매수/매도 추천"이라는 표현 사용 금지
- "공개된 데이터를 기반으로 정리한 내용입니다"라는 고지 항상 포함

## 4. 수익 모델

### 4-1. 구독 수익
| 플랜 | 가격 | 기능 |
|------|------|------|
| 무료 | 0원 | 홈 대시보드, 링크 허브, 채팅, 종목 리스트 열람 |
| 프리미엄 | 29,000원/월 | 종목 상세 10탭, AI 분석 5기법, 실시간 공시 알림, 급등락 알림, 관심종목 무제한, 한국+미국 통합 |

- 장기 할인: 3개월(-10%), 6개월(-15%), 12개월(-20%)
- 가격 근거: 전업투자자 세팅 비용 대비 압도적 저렴, 유료 리서치 서비스(월 5~10만원) 대비 경쟁력

### 4-2. 배너 광고 수익
| 종류 | 가격 | 혜택 |
|------|------|------|
| 인증업체 배너 | 50,000원/20일 | 프리미엄 상단 영역, 큰 배너(300x250), "인증업체" 마크, 클릭 통계 |
| 일반 배너 | 30,000원/20일 | 하단 일반 영역, 텍스트 또는 작은 배너, 클릭 통계 |

- 인증업체: 사업자등록증 업로드 + 관리자 승인 필요
- 일반 배너: 별도 인증 불필요, 면책 고지 표시
- 셀프서비스: 광고주가 직접 등록/결제/관리

### 4-3. 추후 수익
- 크립토 거래소 제휴 링크 (커미션)
- 프리미엄 API 제공 (B2B)
- 교육 콘텐츠 마켓플레이스

## 5. 다국가 확장 전략

### 핵심 아이디어
"같은 템플릿, 데이터 소스만 교체"
- 국가를 추가할 때 코드를 새로 짜는 게 아니라, 데이터 소스 연결 + 언어 번역만 하면 됨
- 한계 비용이 거의 0에 수렴

### 확장 로드맵
```
Phase 1: 한국 (코스피/코스닥) — 현재
Phase 2: 미국 (나스닥/NYSE) — 데이터 소스 이미 준비 (SEC EDGAR, FRED)
Phase 3: 일본 (TSE) / 홍콩 (HKEX) — 한국 투자자들이 많이 거래하는 시장
Phase 4: 크립토 — 별도 브랜드 분리, 24시간 시장, 국경 없음
Phase 5: 글로벌 — 영어 버전, 유럽/남미 시장 추가
```

### 국가 추가 시 필요한 것
1. 해당 국가의 공시 API (한국=DART, 미국=SEC EDGAR, 일본=EDINET)
2. 해당 국가의 거래소 데이터 (한국=KRX, 미국=NYSE/NASDAQ)
3. 해당 국가의 거시경제 API (한국=ECOS, 미국=FRED)
4. 언어 번역 (한국어 → 영어/일본어)
5. 결제 현지화 (한국=토스페이먼츠, 글로벌=Paddle)

### 수익 곱하기 효과
- 국가별 구독료 + 국가별 광고주 = 병렬 수익 구조
- 한 국가에서 성공한 모델을 복제하면 됨

## 6. 결제 전략

### 한국 시장: 토스페이먼츠
- 정기결제 (빌링키 방식)
- PG 수수료가 낮고 한국에서 점유율 높음
- 카드 결제 + 간편결제 지원

### 글로벌 시장: Paddle
- 미국 법인 없이도 글로벌 결제 가능 (Paddle이 판매 대행)
- 세금/VAT 자동 처리
- 한국에서는 토스페이먼츠, 해외에서는 Paddle로 이원화

## 7. 창업자 배경 — 왜 이 사업을 할 수 있는가

### 핵심 역량
- 주식/가상화폐 광고주 허가 콜센터 운영 경험
- 광고주 네트워크 확보 (배너 광고 수익 모델의 핵심)
- 투자자 커뮤니티와의 접점 보유
- 규제 환경 이해 (금융 광고 규제에 대한 실무 지식)

### 경쟁 우위
- 광고주를 직접 모집할 수 있는 채널이 이미 있음
- 개발비 0원 (Claude Code 활용)
- 운영비 최소 (클라우드 SaaS 활용)
- 시장 인사이트: 전업투자자들의 실제 니즈를 알고 있음

## 8. 핵심 설계 결정 기록

### 결정 1: 커뮤니티 페이지 삭제
- **이유**: 별도 커뮤니티 페이지는 채팅을 "무겁게" 만들고, 채팅방 링크 배너 광고와 목적이 충돌
- **대안**: 플로팅 채팅만 유지 (좌하단, 접었다 펼 수 있음)
- **효과**: 가벼운 소통 + 배너 광고와 공존

### 결정 2: AI는 "정리"만 한다
- **이유**: 예측은 틀릴 수 있고, 법적 리스크와 신뢰 문제 발생
- **원칙**: AI는 공개 데이터를 분석 기법별로 정리만 하고, 매수/매도 추천 절대 안 함

### 결정 3: PC 우선 개발
- **이유**: 투자 데이터 터미널은 PC 환경이 주력 사용 환경
- **순서**: PC 완성 → 태블릿 → 모바일 순으로 반응형 대응

### 결정 4: MVP가 아닌 완성품
- **이유**: "절대 MVP라고 생각하지 마" — 처음부터 완성도 있게 만들어야 함
- **원칙**: 기능 축소 없이 전체 기능을 구현하되, 데이터가 없는 부분은 empty state를 깔끔하게

### 결정 5: 개발비 0원 전략
- **이유**: "세계최강의 코딩 AI Claude Code가 있는데 왜 개발비를 쓰냐"
- **방법**: Claude Code로 전체 개발, 인프라는 무료/저가 클라우드 활용
- **절감액**: 외주 개발 시 예상 비용 500~1000만원 → 0원

### 결정 6: Supabase 프로젝트 분리
- **이유**: 기존 POTAL 프로젝트와 테이블명이 겹치면 데이터 손상
- **규칙**: StockTerminal 전용 Supabase 프로젝트(OT-Marketing) 사용, 절대 섞지 않음
