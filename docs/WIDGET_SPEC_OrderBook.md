## 1. 레퍼런스 벤치마크

### 주 레퍼런스 — 키움 영웅문 호가창
- **HTS**: 10단 호가, 매도잔량(좌) · 호가(중앙) · 매수잔량(우) 3열 레이아웃
- **MTS**: 5단 호가, 동일 3열 간소화
- **색상 관례** (한국 MTS 표준):
  - 매도잔량 = 파랑 (#0051CC)
  - 매수잔량 = 빨강 (#FF3B30)
  - 현재가 = 강조 배경 (티얼 또는 노랑)

### 보조 레퍼런스 — 한국투자증권 eFriend
- 거의 동일 레이아웃, 잔량 막대 표시 방식 미세 차이

### 보조 레퍼런스 — 네이버증권 호가탭
- 브라우저 친화적 단순화, 5단만 표시, 잔량 막대 없음

## 2. 현재 상태 (STEP 52 기준)

### 홈 위젯 — 작동 중 (개선 여지 있음)
- 5단 호가 표시 (API는 10단 반환, 위젯에선 slice로 5개만)
- 매도=파랑, 매수=빨강 색상 ✅
- 잔량 막대 구현 ✅
- 현재가 + 등락률 표시 ✅
- 5초 자동 갱신 ✅
- 심볼 고정 (005930 삼성전자) — 심볼 변경 불가 ❌

### 현재 레이아웃 문제
- **2-col 구조**: `[호가 | 잔량]` — 매도/매수 구분이 행 색상으로만
- **키움 표준은 3-col**: `[매도잔량 | 호가 | 매수잔량]`
- 총매도/총매수 잔량 요약 없음
- 심볼 검색 폼 없음

### `/orderbook` 페이지 — **스텁 (차후 STEP에서 교체)**
- `WidgetDetailStub` 기반 더미 테이블 20행
- STEP 53 범위 밖 (후속 STEP로 분리)

## 3. Phase A — STEP 53 범위 (즉시 실행)

### 3-1. OrderBookWidget 레이아웃 3-col 재편
- **Before**: `[호가 | 잔량]` 2-col, 매도는 우측에 파란 막대, 매수는 좌측에 빨간 막대
- **After**: `[매도잔량 | 호가 | 매수잔량]` 3-col
  - 매도 5개 행: 좌측에 매도잔량 표시(우측정렬), 호가 중앙, 우측 칸은 빈칸
  - 매수 5개 행: 좌측 빈칸, 호가 중앙, 우측에 매수잔량 표시(좌측정렬)
  - 잔량 막대: 매도는 좌→우, 매수는 우→좌 방향
- 호가 색상: 매도=파랑(#0051CC), 매수=빨강(#FF3B30), 현재가=티얼 배경

### 3-2. 총 매도·매수 잔량 푸터
위젯 하단에 추가:
```
총 매도 1,234,567  ·  총 매수 2,345,678
[매수 65.5% ██████████░░░░░░░░░░ 34.5% 매도]
```
- 매수 비율 = 총매수 / (총매수+총매수) × 100
- 게이지 바: 매수=빨강, 매도=파랑, 중앙 분리선

### 3-3. 심볼 검색 폼 (우상단 action)
- ChartWidget 패턴 차용
- 6자리 숫자만 받는 인풋 + 이동 버튼
- 형식 불일치 시 경고 없이 submit 무시

### 3-4. href 동적화
- **Before**: `href="/orderbook"` (고정)
- **After**: `href={'/orderbook?symbol=' + symbol}` (현재 심볼 전달)

### 3-5. Phase A 제외 (의도적)
- 10단 표시: 공간 제약 → Phase B (상세 페이지에서 10단)
- 전일대비 호가 색상 그라데이션: 키움 HTS 전용, MVP 후순위
- 잔량 변화 애니메이션: Phase C
- 실시간 웹소켓: 현재 5초 폴링 유지
