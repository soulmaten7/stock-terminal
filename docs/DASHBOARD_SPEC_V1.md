## 1. 핵심 설계 원칙 (2026-04-21 신규)

### 원칙 1. 위젯 vs 사이드바 = "실시간성"으로 결정

| 공간 | 조건 | 이유 |
|------|------|------|
| **위젯** (대시보드 그리드 상시 노출) | 실시간 변동 콘텐츠만 | 안 변하는 것 상시 노출 = 면적 낭비 |
| **사이드바 / 상세 페이지** (클릭 진입) | 정적·편집·디테일 콘텐츠 | 필요할 때만 접근, 넓은 공간 활용 |

### 원칙 2. 대시보드 첫 화면 = "지금 뭐가 일어나는지" 직관 중심
- 시장조사 16/16 = First Fold 최상단 실시간 이벤트 배치 (100% 공통 패턴)
- 초보자 타겟이어도 동일 — Jakob's Law

### 원칙 3. 하드코딩 금지
- 심볼 기본값 = KIS 거래량 1위 동적 결정 (Option B, 2026-04-21 결정)
- 전역 `selectedSymbol` Zustand 스토어 도입 예정 (차트·호가·체결·관심 연동)

### 원칙 4. "공통점은 기준선, 드문 점에서 승부"
- 시장 13~16곳 공통 = 필수 배제 불가 (Jakob's Law)
- 시장 3~7곳만 = 차별화 기회 (예: 테마 위젯, 실시간 채팅 First Fold)

## 4. 보류 사항 (네 결정 대기)

### 4.1 🔄 채팅 UX — 사용자 고민 중 (2026-04-21)

**선택지**:
- **A. 현재 유지** — 대시보드 R3C1 한 칸 (위젯형)
- **B. 우측 고정 사이드바** — LiveScore 방식, 항상 열림
- **C. 플로팅 패널** — 아이콘 클릭 시 열림
- **D. 전용 페이지 중심** — 대시보드 프리뷰만 + `/chat` 상세

**결정에 영향 주는 질문**:
1. **채팅 범위**: 전체 마켓 토크 vs 종목별 토론 vs 혼합
2. **발견성 vs 방해**: 항상 보이게(engagement ↑) vs 필요할 때만(집중력 ↑)
3. **로그인 정책**: 비로그인 읽기+로그인 쓰기 (현재) vs 로그인 필수
4. **모바일 대응**: 모바일에서도 같은 위치? 다른 위치?

**채팅 위치 결정 = 사이드바 구조 전체 결정의 트리거**.

### 4.2 R5C3 빈 자리 채우기

장전 브리핑 제거 + 뉴스 축소로 R5C3 빈 칸 발생.

**후보**:
- A. **종목 뉴스 필터** (selectedSymbol 관련 뉴스만, 실시간)
- B. **52주 신고가·신저가 랭킹** (실시간)
- C. **급등 알림 피드** (거래량/상승률 급변 종목 티커, 실시간)
- D. **빈 공간 유지** (여백 = 밀도 완화)

### 4.3 사이드바 구현 방식

**선택지**:
- 방식 1: **좌측 네비 아이콘 → 별도 페이지 이동** (현재 방식 강화)
- 방식 2: **슬라이드 패널** (대시보드 위로 떠서 열림)
- 방식 3: **우측 고정 사이드바** (LiveScore 방식)

→ §4.1 채팅 결정과 연동 필요.
