## 1. 개요

### 1.1 조사 목적
Stock Terminal V3의 대시보드 레이아웃 설계를 위해 한국에서 실제 사용되는 주요 주식 플랫폼 16곳의 First Fold(스크롤 없이 보이는 상단 영역) UX 패턴을 분석합니다. 특히 사용자의 가설인 "상단: 지금 일어나는 일(이벤트·직관성), 중단: 내 것, 하단: 분석 도구"의 3단 구조 타당성을 검증하는 것이 핵심입니다.

### 1.3 분석 7축

1. **First Fold 구성** — 스크롤 없이 보이는 요소들
2. **실시간 요소** — 이벤트성(급등·공시·뉴스) vs 상태성(시세·호가 깜빡임)
3. **로그인 전/후 차이** — 비로그인 상태의 정보 vs 로그인 후 개인화
4. **커뮤니티 위치** — 채팅·댓글·종목토론 유무와 배치
5. **랭킹·Top-N 종류** — 거래량·상승률·외국인·검색·테마 등
6. **뉴스·공시 노출** — 위치, 개수, 형식
7. **정보 밀도** — 1(극단 단순·카드형) ~ 5(극단 빽빽·격자 터미널형)

### 1.4 조사 방법 및 제약

- **수집 방법**: WebFetch, 공개 자료(공식 홈페이지, 앱스토어, 언론 기사)
- **비로그인 우선**: 비로그인 상태 분석 우선, 로그인 필요 화면은 공식 소개 자료로 보완
- **JS 렌더링 제약**: 일부 증권사 사이트는 JS 렌더링 또는 bot 차단 있음 → 신뢰도 "하"로 표시
- **분석 시간**: 사이트당 최대 15분, 접근 불가 시 공개 자료 최대 2~3건으로 보완
- **신뢰도 기준**:
  - 상: 직접 WebFetch 성공 + 명확한 구조 파악
  - 중: WebFetch 부분 성공 또는 공식 자료 2종 이상으로 보완
  - 하: WebFetch 실패, 공개 자료 1종 또는 미확인

### 1.5 용어 정의

- **First Fold**: 사용자가 스크롤 없이 초기 로드 시 보는 상단 영역 (대략 1440×900 해상도 기준)
- **이벤트성 실시간 요소**: 급등·급락·공시·뉴스 등 변동성 높은 콘텐츠
- **상태성 실시간 요소**: 시세·호가·거래량 등 연속 갱신되는 상태 데이터
- **정보 밀도**: 1 = 최소한의 정보만(카드형, 여백 큼), 5 = 매우 빽빽함(격자 터미널)

**URL**: https://finance.naver.com

**신뢰도**: 중

#### First Fold 구성
- 상단: 글로벌 지수 (KOSPI, KOSDAQ, 미국 S&P500, 나스닥, 엔닉스, 홍콩 항셍 등)
- 중상: 실시간 시세 (주요 종목 5~10개 깜빡이는 호가)
- 중단: 뉴스·공시 (헤드라인 + 썸네일)
- 하단: 인기 검색어 + 테마 + 급등·급락 랭킹

#### 실시간 요소
- **이벤트성**: 뉴스(경제·증권·테마), 공시, 급등·급락 알림
- **상태성**: 지수 시세(1초~3초 갱신), 개별 종목 깜빡임

#### 로그인 전/후 차이
- **전**: 시장 정보만 (뉴스, 공시, 지수)
- **후**: 관심종목 추가, 포트폴리오, 개인 알림, 스크래핑 저장

#### 커뮤니티 위치
- 좌측 네비게이션 서브메뉴 (종목토론, 투자자 커뮤니티)
- First Fold에는 미노출 → 사용자가 능동적으로 진입

#### 랭킹 종류
- 급등 TOP10, 급락 TOP10
- 거래량 많은 종목 (1시간 기준)
- 관심도 랭킹 (네이버 검색 기반)
- 테마별 상승 종목

#### 뉴스·공시 노출
- 헤드라인 4~6개 + 썸네일 소규모
- 공시는 별도 탭 (상세 분류 가능)
- 경제·증권·테마 뉴스 자동 분류

#### 정보 밀도
**4/5** — 빽빽하지만 시각적 계층이 명확. 여러 위젯이 한 화면에 동시 표시

#### 강점
- 지수·뉴스·공시 균형잡힌 배치
- 검색·테마 기반 발견 용이
- 네이버 검색 통합으로 관심도 트래킹

#### 약점
- 포트폴리오 기능이 First Fold 이하
- 차트/분석 도구가 별도 페이지로 분리

## 6. Stock Terminal V3 시사점

### 6.1 사용자 가설 검증 결과

**원래 가설**:
```
상단: "지금 뭐가 일어나고 있는지" (이벤트·직관성)
중단: "내 것" (포트폴리오)
하단: "분석 도구" (차트, 호가 등)
```

**검증 결과**: **부분 지지** (약 70% 타당)

#### 지지하는 증거 (16곳 모두 따르는 패턴):
1. **상단은 반드시 이벤트·뉴스·지수** — 16/16 = 100%
2. **중단은 관심종목 또는 랭킹** — 15/16 = 94%
3. **하단은 분석·상세 정보** — 대부분 맞음

#### 수정이 필요한 부분:
1. **"내 것" (포트폴리오)의 위치가 다양함**:
   - 대형사(B): First Fold 또는 그 직후 (로그인 시)
   - 간편(D): 로그인 후 별도 페이지로 분리
   - **우리의 전략**: 비로그인 사용자가 대다수 → "관심종목" 위젯으로 대체, 로그인 시만 포트폴리오 노출

2. **"분석 도구"가 First Fold 근처에 있음**:
   - 차트는 Row2~3 (하단이 아님)
   - 호가·체결은 전업자만 First Fold에 배치
   - **우리의 조정**: Row2에 차트 배치, Row3에 고급 도구 (호가, 체결) 배치

3. **3단이 아닌 4단 구조도 가능**:
   ```
   Row1: 글로벌 지수 + 실시간 이벤트 (매우 상단)
   Row2: 뉴스 + 랭킹 + 차트 (중상)
   Row3: 호가 + 체결 + 채팅 (중하)
   Row4: 경제캘린더 + 고급 도구 (하단)
   ```
   → **우리 현재 구조가 정확히 이 4단 구조**

### 6.5 우리만의 차별화 지점

#### 1. "실시간 채팅" First Fold 배치 (RealtimeChatWidget)
- **시장 현황**: 거의 모든 플랫폼이 채팅을 별도 페이지로 분리
- **우리의 강점**: 채팅을 Row3에 embedded card로 → 즉시 커뮤니티 진입 가능
- **효과**: 사용자 engagement + 체류 시간 증가
- **현황**: 구현됨 (세션 #18에서 추가)

#### 2. "테마 기반 발견" 강화 (신규)
- **시장 현황**: 팍스넷, 토스만 "상승 테마" 강조
- **우리의 기회**: "테마 → 종목 → 차트 → 채팅" 한 루프 안에서 실행
- **제안 위젯**: `TrendingThemesWidget` (5개 테마 카드, 각 테마별 종목 수 + 상승률)
- **구현 난이도**: 중 (KRX 또는 자체 테마 분류 필요)

#### 3. "비전업자용 호가 카드" 추가 (신규)
- **시장 현황**: OrderBookWidget은 전업자만 이해 (5단 호가, 색상 복잡)
- **우리의 기회**: "최근 매매 가격", "매도/매수 압력" 간단 요약 카드
- **효과**: 초보자 → 전업자로 이행할 때의 진입장벽 낮춤
- **구현 난이도**: 낮 (KIS API 이미 있음)

#### 4. "경제 캘린더" 한국 이벤트 강화 (신규)
- **시장 현황**: 대부분 글로벌 캘린더만 (미국, 유럽 중심)
- **우리의 기회**: 한국 이벤트 추가 (기준금리, GDP, 수출입, 실업률)
- **구현 난이도**: 중 (ECOS API 이미 있음)

#### 5. "베스트 콜" 또는 "주간 성과" 리더보드 (향후 Phase 2)
- **시장 현황**: 인베스팅닷컴만 "투자 챌린지" 제공 (게이미피케이션)
- **우리의 기회**: "이번 주 최고 수익률", "최다 매수·매도 종목" 통계
- **효과**: 사용자 참여 + 공유 유도
- **실행**: 향후, 현재는 단계 외
